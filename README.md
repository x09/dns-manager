# Диспетчер DNS для Samba DC — версия 2.0

Графическая утилита для Linux — аналог **Microsoft DNS Manager**.
Подключается к DNS-серверу, встроенному в контроллер домена **Samba DC**,
по протоколу MS-DNSP (DCERPC) — тому же, что использует оснастка Windows
и `samba-tool dns`.

## Новое в версии 2.0

- **Служебные разделы (вложенные папки) зон.** Зоны в дереве раскрываются
  на вложенные узлы — `_sites`, `_tcp`, `_udp`, `DomainDnsZones`,
  `ForestDnsZones`, `dc`, `domains`, `gc`, `pdc` и т.д. — как в
  Microsoft DNS Manager. Содержимое папок подгружается по мере раскрытия.
- Папки отображаются и в правой панели; двойной щелчок открывает папку.
- Записи (в т.ч. SRV в `_ldap._tcp.<site>._sites.DomainDnsZones` и другие)
  можно просматривать, создавать, изменять и удалять в любой папке зоны —
  имена в диалоге задаются относительно текущей папки.

## Возможности

- Просмотр, создание и удаление прямых и обратных (reverse) DNS-зон
  (зоны создаются как основные, интегрированные в Active Directory).
- Создание, изменение и удаление записей: **A, AAAA, CNAME, MX, PTR, SRV, TXT**
  (записи NS и SOA отображаются в режиме «только чтение»).
- Автоматическое создание связанной PTR-записи при добавлении A-записи
  (по флажку в диалоге, если существует подходящая обратная зона).
- Интерфейс в стиле MS DNS Manager: слева дерево сервера, зон и папок,
  справа список записей выбранной папки; контекстные меню, F5 — обновление.
- При запуске запрашиваются сервер, логин и пароль. Сервер и логин
  сохраняются в `~/.config/samba-dns-manager/config.json`
  (пароль не сохраняется).

## Требования (ОС Альт)

- Python 3.12 с tkinter:

  ```
  # apt-get install python3-modules-tkinter
  ```

- Python-биндинги Samba (модуль `samba.dcerpc.dnsserver`):

  ```
  # apt-get install python3-module-samba
  ```

  Модуль также входит в состав пакетов `samba-dc` / `samba-client`.

Устанавливать что-либо через pip не требуется.

## Запуск

```
chmod +x dns-manager.py   # один раз
./dns-manager.py
```

или `python3 dns-manager.py`.

## Подключение

- **Сервер** — имя или IP-адрес контроллера домена Samba.
- **Пользователь** — учётная запись с правами на управление DNS
  (член группы `Domain Admins` или `DnsAdmins`). Поддерживаются формы
  `user`, `DOMAIN\user`, `user@realm`.
- **Пароль** — запрашивается при каждом запуске.

## Состав файлов

```
dns-manager.py          — исполняемый файл (точка входа)
dnsmgr/
    __init__.py
    backend.py          — работа с сервером по MS-DNSP (RPC), узлы/папки зон
    config.py           — сохранение сервера/логина
    dialogs.py          — диалоги подключения, зон и записей
    mainwindow.py       — главное окно (дерево зон и папок)
```

## Примечания

- Создание обратной зоны: достаточно указать ИД сети
  (например `192.168.1` или `10.0.0.0/16`) — имя зоны
  `*.in-addr.arpa` будет построено автоматически. Поддерживаются и
  IPv6-зоны (`*.ip6.arpa`) — укажите префикс, например `2001:db8::/32`.
- При добавлении PTR-записи в корень обратной зоны можно ввести полный
  IP-адрес — утилита сама вычислит относительное имя записи.
- Изменение записи выполняется атомарно на сервере
  (старое значение заменяется новым одним RPC-вызовом).
- После обновления (F5) выбор возвращается к корню текущей зоны.
